# StudyHub — docs/

Public-facing documentation lives here.

Detailed planning, roadmaps, release logs, security runbooks, and product-strategy documents are kept in `docs/internal/` and are **not** checked into the public repository (see `.gitignore`). They are private working documents for the StudyHub core team.

If StudyHub is ever open-sourced in the future, individual documents from `docs/internal/` may be reviewed, redacted where appropriate, and promoted back to this folder. Nothing is exposed by default.

For API reference, contribution guide, or any publicly useful material, look for files alongside this README.
